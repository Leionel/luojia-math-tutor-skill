后端 Agent 架构：RAG 如何融入 LangGraph 状态机？

我们要把语义检索 (RAG) 和 LangGraph 状态机结合。请选择你期望 RAG 在工作流中的触发机制：

方案 A：前置隐式检索 (Implicit Retrieval Node)
[用户输入]
  │
  ▼
[意图识别] (Intent Detect)
  │
  ▼
[自动 RAG] (后台静默检索知识库及课件)
  │
  ▼
[LLM 思考 & 验算] (携带检索到的 context 进行推理与 Sandbox 执行)
  │
  ▼
[输出结果]
🔍 特点：用户发送问题后，系统立刻调用 Embedding 进行匹配并注入 Prompt。大模型在动笔前就拥有了知识库，反应速度快，开发简单。
方案 B：主动工具调用 (Explicit Tool Node)
[用户输入]
  │
  ▼
[LLM 启动分析] (主控 Node)
  ├──► [需知识库] ──► [Tool: Search_Knowledge] ──┐
  ├──► [需验算] ──► [Tool: Sandbox_Exec] ────┼─► [LLM 再决策]
  └──► [已完成] ──► [生成最终回复] ◄──────────┘
🔍 特点：大模型自己决定是否需要去翻书（调用 RAG 工具）。如果问题很简单，就不去查，如果发现复杂，就发指令检索，极具 Agent 自主性。
