# 珞珈数智助教省赛智能体赛道延伸说明

## 1. 总体判断

现有项目适合包装为“中国移动杯”2026 江苏生成式人工智能创新大赛的“智能体”赛道作品，但不建议只以“数学教学工具”来表达。

更合适的定位是：

> 珞珈数智助教是一个面向 STEM 教育的可验证学习智能体，以大学数学作为首个落地场景，构建“生成式 AI + 符号验证 + 错因记忆 + 自适应练习”的学习闭环。

这样处理后，数学不是项目边界，而是一个高价值、高难度、可验证的示范场景。项目核心价值从“会讲数学题”提升为“能在严谨推理场景中规划、调用工具、验证结果、记录学习状态并持续引导训练的教育智能体框架”。

## 2. 为什么不能只讲数学教学

如果只讲“AI 数学助教”，项目容易被评委理解为普通教育类应用，存在三个问题：

- 场景显得窄：只服务数学学习，产业想象空间不够。
- 技术显得浅：容易被误解为调用大模型回答题目。
- 商业显得弱：C 端学生工具的付费和推广路径不够有说服力。

因此，省赛叙事应从“教学功能”上升到“智能体能力”：

- Planning：识别学科、题型和用户意图，决定教学路径。
- Tool Use：调用 SymPy/Python 对关键步骤进行符号或数值验算。
- Memory：记录会话、尝试、错因事件和轻量掌握度。
- Adaptive Feedback：根据模式、错因和掌握度调整提示层级。
- Knowledge Grounding：基于本地结构化知识库给出课程依据。

## 3. 推荐项目定位

### 3.1 项目名称

建议使用：

> 珞珈数智助教：面向 STEM 教育的可验证学习智能体

### 3.2 副标题

建议使用：

> 以大学数学为首个落地场景，构建“生成式 AI + 符号验证 + 错因记忆 + 自适应练习”的学习闭环。

### 3.3 一句话介绍

> 珞珈数智助教通过大模型完成自然语言理解、教学规划和表达生成，通过本地数学知识库提供课程依据，通过 SymPy/Python 工具验证关键推导步骤，并结合错因记录和自适应练习形成持续学习闭环。

## 4. 五个核心延伸方向

### 4.1 从单题答疑延伸到学习闭环

当前 MVP 已经具备对话、验算、错因记录和会话保存能力。下一阶段最值得强化的是学习闭环：

```text
用户提交题目或步骤
-> 智能体识别意图和学科
-> 检索本地知识库
-> 调用符号工具验算
-> 诊断错因
-> 调整提示层级
-> 生成类似题
-> 复练并更新掌握度
```

省赛表达建议：

> 本项目不是一次性答题工具，而是错因驱动的自适应学习智能体。它能根据学生当前错误、连续出错情况和历史掌握度，动态调整引导深度，并推荐下一步练习。

可落地功能：

- 错因事件记录。
- 轻量 mastery 记录。
- 提示层级策略。
- 类似题生成。
- 右侧学习状态面板。

### 4.2 从数学教学延伸到可验证推理

大学数学是一个高强度验证场景，适合展示智能体的严谨推理能力。它要求：

- 公式和符号推导准确。
- 步骤可以被检查。
- 错误可以被定位。
- 解释不能只靠流畅语言。

因此，项目可以包装为：

> 面向严谨推理任务的 Verification Agent。

当前数学场景中使用 SymPy 验证积分、求导、矩阵、行列式、条件概率等。未来可迁移到：

- 物理：公式推导、量纲检查、数值仿真。
- 电路：节点方程、矩阵求解、频响分析。
- 控制：传递函数、稳定性判定。
- 统计：假设检验、分布计算、置信区间。
- 工程课程：力学、信号系统、优化基础。

省赛表达建议：

> 数学是首个落地场景，底层能力是“生成式 AI + 确定性工具验证”的通用学习智能体框架。

### 4.3 从知识库延伸到课程级智能体

学校赛版本强调 10000+ 数学语料，这一点很好，但省赛里要避免只说“我有很多数据”。更好的表达是：

> 每门课程都可以被抽象为“知识库 + 错因库 + 工具链 + 练习库 + 教学策略”的课程级 Agent。

大学数学课程级智能体包含：

- 教材定义与定理。
- 典型题型。
- 常见错因。
- 可调用验算工具。
- 类似题与复练机制。
- 提示层级策略。

未来扩展时，不是重新做一个产品，而是为新课程替换或补充：

- 课程知识库。
- 工具调用规则。
- 错因规则。
- 练习生成模板。

这能体现项目的平台化潜力。

### 4.4 从学生端延伸到教师端和机构端

只做学生端会让商业空间显得偏小。建议在计划书和 PPT 中加入教师端/机构端路线。

学生端价值：

- 随时答疑。
- 分步引导。
- 错因诊断。
- 类似题训练。

教师端价值：

- 班级错因热力图。
- 知识点薄弱统计。
- 作业步骤诊断。
- 个性化练习推荐。

机构端价值：

- 教培机构智能答疑 API。
- 在线教育平台练习引擎。
- 高校公共课智能助教服务。

商业路径可以写为：

```text
C 端免费/低价学习工具
-> 高校课程试点
-> 教培机构和在线教育平台 API
-> 课程级智能体解决方案
```

### 4.5 从文本对话延伸到多模态学习

学校赛材料中已有动态可视化方向，这是省赛的加分项。建议不要只说“会画图”，而是表达为：

> 智能体根据知识类型自动选择输出形态，包括文本、公式、图像和练习任务。

可视化示例：

- 梯度：等值线 + 梯度向量。
- 积分：面积/体积可视化。
- 概率分布：密度曲线和阴影面积。
- 线性代数：矩阵变换前后的图形变化。

注意：多模态可视化建议作为展示加分项，不要抢走“学习闭环”和“可验证推理”的主线。

## 5. 推荐省赛主线

建议采用三阶段路线：

### Phase 1：大学数学 Tutor Agent

已实现或正在完善：

- 高数、线代、概率论三门课程。
- 流式对话。
- 本地知识库检索。
- SymPy 后台验算。
- 错因事件记录。
- DeepSeek/OpenAI-compatible 模型接入。

### Phase 2：课程级学习闭环

下一阶段重点：

- 掌握度更新。
- 自适应提示层级。
- 错因复盘卡片。
- 类似题生成。
- 复练路径推荐。

### Phase 3：STEM 可验证学习智能体平台

未来扩展：

- 物理、电路、统计、控制等课程。
- 教师端学习分析面板。
- 教培机构 API。
- 多模态可视化学习组件。

## 6. 项目计划书延伸写法

### 6.1 项目概述

建议重点写：

- 背景：生成式 AI 正在进入教育场景，但严谨推理类课程仍存在幻觉、剧透、不可追踪等问题。
- 目标：构建一个会规划、会验算、会诊断、会复练的可验证学习智能体。
- 场景：以大学数学为首个落地场景，服务高校公共基础课、考研基础训练和教培答疑。
- 创新：神经符号双引擎、启发式交互、错因记忆、自适应练习、结构化知识库。

### 6.2 技术方案

建议按照智能体架构写：

```text
用户输入
-> Intent Router
-> Knowledge Search
-> Verification Tools
-> Misconception Detector
-> Hint Policy
-> Prompt Builder
-> LLM Streaming Response
-> Memory Update
```

可以使用五层架构：

- 交互层：Next.js 工作台、三栏界面、KaTeX、SSE。
- 编排层：意图识别、学科识别、模式路由。
- 知识层：结构化数学知识库。
- 工具层：SymPy/Python 验算。
- 记忆层：会话、消息、尝试、错因、轻量掌握度。

### 6.3 应用落地

如果没有真实大规模数据，不要编造。可以写：

- 已完成本地 MVP。
- 已支持高数、线代、概率论基础题型。
- 已完成若干端到端验收题。
- 计划在高校课程学习小组、期末复习场景中试点。

可规划的成效指标：

- 可验算题型正确识别率。
- 错因识别命中率。
- 平均响应时间。
- 复练正确率提升。
- 学生主动完成率。

### 6.4 竞品分析

建议对比四类产品：

| 类型 | 代表 | 局限 | 珞珈数智助教优势 |
|---|---|---|---|
| 通用大模型 | ChatGPT、DeepSeek、文心一言 | 数学计算可能幻觉，缺少学习状态追踪 | 符号工具验算 + 错因记忆 |
| 搜题 APP | 作业帮、小猿搜题 | 直接给答案，弱化思考过程 | 启发式分步引导 |
| 科学计算工具 | WolframAlpha、Mathematica | 门槛高，偏计算而非教学 | 自然语言交互 + 隐形工具调用 |
| 普通 AI Tutor | 各类教育聊天机器人 | 多为文本问答，缺少确定性验证 | LLM + Knowledge + Verification + Memory |

## 7. PPT 延伸结构

建议控制在 16 页左右：

1. 封面：面向 STEM 教育的可验证学习智能体。
2. 行业痛点：通用 AI 在严谨推理教育中的失效。
3. 为什么选择大学数学：高频、刚需、强验证。
4. 产品方案一句话：知识库 + 工具验算 + 错因记忆 + 自适应练习。
5. Agent 总架构图。
6. Planning：意图识别与模式路由。
7. Tools：SymPy/Python 后台验算。
8. Memory：会话、尝试、错因、掌握度。
9. Knowledge：结构化数学知识库。
10. Demo 1：积分错误纠正。
11. Demo 2：链式法则、行列式、互斥独立。
12. 学习闭环：诊断 -> 提示 -> 类似题 -> 复练。
13. 产品界面截图。
14. 竞品分析矩阵。
15. 落地路径与商业模式。
16. 未来规划与总结。

## 8. 表述风险与替代表达

省赛材料要避免过度夸张。建议替换如下：

| 不建议写法 | 推荐写法 |
|---|---|
| 国内首款 | 面向高校数学场景的可验证学习智能体 |
| 0 幻觉 | 对可符号化步骤进行后台验证，显著降低计算幻觉 |
| 100% 验算 | 对 MVP 覆盖题型进行确定性工具校验 |
| 99.9% 准确率 | 在自建验收题集上达到较高验证准确率，后续持续扩大评测 |
| 已商业化 MaaS | 未来可向高校和教培机构提供 Tutor Agent API |
| 已构建向量数据库 | 已构建结构化知识库，后续可扩展向量检索 |

## 9. 最终建议

本项目最有竞争力的方向不是“大而全教育平台”，而是：

> 以大学数学为首个硬核场景，验证一个可规划、可验算、可记忆、可复练的学习智能体框架。

省赛材料应围绕三句话展开：

1. 大学数学是检验 AI 严谨推理能力的高价值场景。
2. 珞珈数智助教通过 LLM + 符号工具 + 本地知识库 + 错因记忆解决通用 AI 的幻觉和剧透问题。
3. 该框架未来可迁移到更多 STEM 课程，并服务高校课程助教、在线教育平台和教培机构。
