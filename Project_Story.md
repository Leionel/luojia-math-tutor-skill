# 珞珈数智助教 (Luojia Math Tutor) —— 从单体“问答”到多智能体“名师”的架构演进

## 1. 痛点：为什么传统的 AI 不适合做数学助教？
在项目初期，我们遇到的大模型在教育场景中的通病：
- **无底线的“溺爱”**：学生只要多要几次，AI 就会把正确答案全盘托出，变成纯粹的“代写工具”。
- **角色的“精神分裂”**：模型前一秒还在扮演“启发式引导的老师”，后一秒又因为内部验证逻辑变身“算题机器”，直接输出大段推理过程，甚至把内部的中间变量暴露给学生。
- **数学幻觉**：大模型自行做积分和矩阵运算时极易出错，误人子弟。

## 2. 破局：Role-based Multi-Agent 与按需调度
为了解决上述问题，我们将单体大模型拆分为四个高度解耦、各司其职的角色，并由 LangGraph 条件状态机调度。角色边界仍然清晰，但不再让所有 Agent 每轮固定串行运行：

1. **Learning Planner（宏观规划师）**：常见请求由本地规则结合 BKT 历史即时生成教学目标，复杂或低置信度场景才升级到远程模型。
2. **Policy Router（动作路由）**：本地优先判定“提示、反问、讲解、复习或出题”；语义歧义较大时才调用 Policy LLM fallback。
3. **Verifier（裁判员）**：积分、求导等明确步骤优先由受控 SymPy 函数确定性校验；复杂证明或本地无法判断的内容才调用 Verifier LLM。
4. **Teacher（温柔的名师）**：只接收 Verifier 的裁判结果与 Planner 的教学目标，不负责算题。它只负责把生硬的结论转化为温柔、启发式的中文，循循善诱，**彻底堵死大模型在解题时“不小心漏出答案”的可能性**。

系统收到问题后会先在本地生成一段安全且相关的开场，再并行读取 Memory/BKT、执行 BM25 和必要的 SymPy 检查。普通请求通常只需要一次 Teacher 流式调用；高风险请求最多经过 `Verifier -> Teacher` 两次模型调用。这个变化不是删除多智能体，而是把“多角色职责”从固定流水线升级为按风险调度。

## 3. 核心突破：BKT (贝叶斯知识追踪) 防作弊引擎
为了量化学生的学习情况，我们摒弃了简单的“答对加分、答错扣分”机制，引入了自适应学习领域的国际前沿算法——**Bayesian Knowledge Tracing (BKT)**。

通过动态的概率公式 $P(L_{t} | obs) = \frac{P(L_{t-1}) \cdot P(obs|L)}{P(obs)}$：
- **防骗分机制**：如果系统发现学生是在高度提示（Hint Level 3）下做对的，系统的 `Guess` 参数会极高，导致学生的知识掌握度雷达图几乎不会上升。
- **自适应练习**：只有当学生独立做对，BKT 雷达图才会显著扩张，并据此决定是否结束该知识点的辅导。

## 4. 黄金标准：LuojiaMathBench V8 评测基准
传统的大模型数学评测基准（如 GSM8K, MATH）只关注“最后结果算得对不对”。这对于教育 AI 是毫无意义的。

为此，我们构建了专属的 `LuojiaMathBench`。这个数据集打破了常规，要求模型必须具备：
- **Error Localization (错因定位)**：不仅仅要看出对错，必须能指出学生究竟在第几步公式写错了。
- **Pedagogical Alignment (教学对齐)**：评测中严格考核大模型是否克制、是否避免了直接给出答案（Direct Answer Leak）。
- 附带了丰富的 `student_profile`（学情画像）和 `learning_objective`（教学目标）字段，全方位模拟真实的线上 1V1 教学场景。

## 5. 总结
本项目不仅仅是一个“接了 ChatGPT API”的网页套壳，而是深入挖掘了**自适应教育（Adaptive Learning）**理论与 **Multi-Agent 协同工程** 的结合点。通过底层强校验、架构强解耦、数据强基准，真正打造了一个有记忆、懂策略、不泄题的智能数学私教。
