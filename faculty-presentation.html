引言：为什么需要可验证学习智能体

通用大模型的两大死穴

1. 直接泄露答案

大模型基于自回归概率引擎，天然倾向于"给出最终答案"以讨好用户，剥夺学生思考机会。

2. 逻辑幻觉

复杂微积分与线性代数推导中，中间一步算错则后续全部崩溃，且模型往往无法自检。

传统工具断层

拍照搜题 App

依赖静态题库完全匹配，无法批改非标自写推导，沦为"抄写工具"。

科学计算工具

WolframAlpha / Mathematica 计算严密，但交互门槛极高，缺乏教学引导。

我们的核心定位

不直接"给答案"，而是"搭脚手架"；不只是"会说"，更是"会验"。

数学基础：BKT 与符号验证

贝叶斯知识追踪 (BKT)

基于隐马尔可夫模型的数学原理，实现动态的掌握度追踪。

$$\underbrace{P(L_{n+1})}_{\text{下次掌握度}} = \underbrace{P(L_n)}_{\text{当前掌握度}} \cdot (1 - \underbrace{P(S)}_{\text{失误率}}) + (1 - P(L_n)) \cdot \underbrace{P(G)}_{\text{猜测率}}$$

P(L)

掌握概率

P(T)

学习迁移率

P(S)

失误率

P(G)

猜测率

SymPy 符号验证

利用 Python 符号数学库，提供确定性的推导校验能力。

from sympy import symbols, diff, Eq, simplify

x = symbols('x')

student_expr = diff(x**3 + 2*x, x)

correct_expr = 3*x**2 + 2

assert simplify(student_expr - correct_expr) == 0

已实现的验证器清单：

  • ✅ 等价性比较 (simplify)
  • ✅ 线性代数矩阵运算 (Matrix, det)
  • ✅ 极限验证 (limit, series)
  • ✅ 解方程/方程组 (solve, linsolve)
